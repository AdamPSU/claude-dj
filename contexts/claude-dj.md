# ClaudeDJ core loop

The user hears music.
The backend watches reactions.
Redis remembers what happened.
Vector search finds possible next songs.
The DJ ranks them using recent context.
Then the player plays the next song.

Features
Phase 1:

Model explicitly says what songs is playing next
Facial recognition to read reactions
Song embedding
Rec. engine
Vector search and ranking
CLI

Phase 2:

Session history import

QoL:

Session history
Ability to verbally request songs
